# Contributing

Thanks for improving NAT Reality Bridge.

## How To Contribute

1. Open an issue describing the change.
2. Keep examples generic and reproducible.
3. Prefer small pull requests.
4. Test documentation commands where possible.
5. Explain whether a change is verified or theoretical.

## Do Not Commit Secrets

Never submit:

- Production server addresses
- SSH credentials
- Proxy credentials
- Reality private keys
- Production UUIDs
- Live VLESS links
- Personal provider account details

Use placeholders such as:

```text
CHANGE_ME
PUBLIC_HOST
PUBLIC_PORT
PUBLIC_KEY
SHORT_ID
REDACTED
```

Pull requests containing real node information should be rewritten before review.
